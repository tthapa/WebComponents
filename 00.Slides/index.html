  • A Tectonic Shift for Web Development

    • making reusable widgets

Understanding Web Components

  • Lets take a look at existing element, the video tag
Neat. But, what is happening behind the scenes By default, the browser hides all the verbose code so you don’t need to see it or worry about writing it when you want to place a video. Web Components allows you to create elements in this manner. For example, for a slide show we can use 
                    <slide-show transition="fade">
                    <slide src="slideone.jpg" thumb="slideone_thumb.jpg" caption="Look at this image">
                    <slide src="slidetwo.jpg" thumb="slidetwo_thumb.jpg" caption="Look at this other image">
                    </slide-show>

What Makes Web Components

Custom Elements

  • Your own custom HTML tags and elements
  • have their own scripted behaviour and CSS styling
  • Create your own from scratch or extend existing element
  • A Cool Component
    • http://html5-demos.appspot.com/gangnam

Templates

  • Inert HTML chunks (not live until you say so)
    • is a mechanism for holding client-side content that is not to be rendered when a page is loaded but may subsequently be instantiated during runtime using JavaScript

Shadow DOM

  • DOM & style encapsulation
  • Each instance is its own little self contained world
  • http://code.tutsplus.com/tutorials/intro-to-shadow-dom--net-34966

HTML Imports

  • Package and Share HTML/CSS/JS
  • “require()” for the web

Why Web Components
